# glossary.md（用語集）

- 対象: `~\Projects` 配下の横断ルール・ドキュメントで用いる共通用語
- 位置づけ: 用語の短い語義と「正（定義の所在）」への案内を一元管理する。詳細な定義は各正ᴳに置き、本書は参照で引く（CR-064）
- 更新: 用語の追加・改称・削除は提案 → 承認後に反映し、正ᴳの所在も更新する（CR-005）
- 旧称併記（「旧〇〇」）は改称直後の移行措置。棚卸し（CR-009）で旧称の残存がゼロと確認できたら削除する

## 記法（定義済み用語の目印）

- ᴳ は初出を示す記号ではなく、用語集の定義に従う用法と一般的な用法を区別する意味標識である
- 定義済み用語を定義された意味で使う場合は、文脈上明らかな場合を含め、全出現箇所で語尾に上付き ᴳ（U+1D33 MODIFIER LETTER CAPITAL G）を付す（例: タスクᴳ）。トークン削減や見た目を理由に省略しない
- 同じ語を一般的な意味で使う場合、用語名そのものを説明・引用する場合、パス・ファイル名等をリテラルとして示す場合は付けない
- 既存文書は棚卸し（CR-009）のタイミングで付与・点検する
- ᴳ が付く語は本書に定義がある語に限る。`03_Checks` の glossary-terms チェックは「ᴳ 付き語が用語集にあるか」だけを検査し、付与漏れは機械判定できないため書き手が点検する（CR-067）
- ᴳ の上付き表示は閲覧環境の OS 既定フォント（Segoe UI・macOS システム・Noto/DejaVu 等）に依存する。ウェブフォントは同梱しない

## セッション進行

- 開始手続き: セッション開始時に毎回固定で行う、ルール確認と引継ぎ読込。合意は不要。正ᴳ: `session-phases.md`
- フェーズ: インプット / ステップᴳ / アウトプット / 完了条件 を持つセッション進行の単位。独立した成果物と固有の完了条件（依頼者合意）を持つ。正ᴳ: `session-phases.md`
- ステップ: フェーズᴳ内部の手順（旧「サブステップ」）。フェーズᴳの単一アウトプットへ収束する。正ᴳ: `session-phases.md`
- チェックポイント: 工程完了、重要判断の確定、一貫した実装単位の完了、検証完了など、別ツールまたは別セッションが再開できる意味のある区切り。worklogᴳ追記とresumeᴳ更新の単位。正ᴳ: 本書
- タスク: 1つのゴールを達成する作業のまとまり。複数セッションにまたがることができ、フェーズᴳより粗い。正ᴳ: CR-047 / CR-049
- タスクスラッグ: タスクᴳを識別する短い文字列。ブランチを使う場合はブランチ名の記述部、使わない場合はタスクᴳ名から定め、チャット名・task-recordsᴳディレクトリ名で共通に用いる。正ᴳ: `session-phases.md`（フェーズᴳ 1）

## 成果物・記録

- task-records: リポジトリ内開発のタスクᴳ別md成果物（タスクᴳ定義・要件・基本／詳細設計・検証対応表）を収める `docs/task-records/<タスクスラッグ>/`。完了後は書き換えない時点スナップショット。正ᴳ: `session-phases.md`
- 正（source of truth）/ 正の一元化: 同一参照スコープ内における情報の唯一の基準所在。自己完結のため別スコープへ複製する場合は各スコープの正を定め、同期関係を参照可能な上位階層で管理する。正ᴳ: CR-037 / CR-064
- handover / worklog / resume: 継続作業ログ。worklog は重要差分と証跡、resume は現在の再開地点、handover は節目の区間要約を持ち、同じ説明を重複記載しない。正ᴳ: CR-052 / CR-054〜CR-058
- DoD（完了の定義）: タスクᴳ種別ごとの必須成果物と必須検証。適用経路とともにタスクᴳ定義へ記録し、プロジェクト作業ではworklogᴳ、横断作業では最終報告へ消込形式で記載する。正ᴳ: CR-065 / `session-phases.md`
- 検証待ちキュー: 実施できない必須検証を記録し解消時に消し込むもの。プロジェクト作業では `resume.md`、横断作業では最終報告へ記載する。正ᴳ: CR-066

## 設計・品質

- 受入条件 / AC-ID: 要件充足を客観判定する基準と、その識別子（AC-NN）。タスクᴳの完了条件のTC-IDᴳへ対応し、自動テストまたは手動検証の元になる。正ᴳ: `session-phases.md`（フェーズᴳ 2）
- TC-ID: タスクᴳの完了条件を識別するID（TC-NN）。AC-IDᴳまたは直接検証へ対応し、最終状態と証跡を消し込む。正ᴳ: `session-phases.md`
- 受入条件カバレッジ: 全受入条件ᴳ（AC-IDᴳ）に自動テストまたは再現可能な手動検証が割り当てられた割合。100%を必須とする。正ᴳ: `session-phases.md`（フェーズᴳ 5）
- 判断レンズ / reviewer: reviewer は CR-030 の品質ゲートを独立監査し、複数の判断レンズは高リスク判断のトレードオフ確認に使う。正ᴳ: CR-030 / `~\Projects\02_Roles\README.md`
