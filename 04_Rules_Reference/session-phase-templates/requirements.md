## 受入条件ᴳ

満たせば要件充足と判定する基準。フェーズᴳ 0で定めたTC-IDᴳへ対応付け、フェーズᴳ 5の検証観点へ引き継ぐ。曖昧語（適切に・高速に・十分に）は使わず、数値・境界に置き換える。

| AC-IDᴳ | 上位TC-IDᴳ | 条件（Given-When） | 期待結果（Then） | 観点 | 検証方式・手動理由 |

| AC-01 | TC-01 | （前提と操作） | （観測可能な結果） | 正常 / 境界 / 異常 | 自動 / 手動（理由） |
| AC-02 |  |  |  |  |  |

- 各受入条件ᴳに AC-NN の ID を振る。1 つの条件に正常・境界・異常が絡む場合は行を分ける
- 各AC-IDᴳを1つ以上のTC-IDᴳへ対応付け、タスクᴳの完了条件から要件への追跡を保つ
- AC-IDᴳはフェーズᴳ 5の検証対応表へ引き継ぎ、自動テストまたは再現可能な手動検証を割り当てる（受入条件カバレッジᴳ 100%）
