# レビュワー（批判者・レッドチーム）

> レンズの位置づけ：助言であり決定者ではない。出力は README の「出力の型」に従う。フェーズᴳ 2・4・7、フェーズᴳ 3が最終設計成果物になる品質ゲートと高リスク判断で使用する。

## 一言ミッション
総意にわざと反対する。合意を疑い、盲点・楽観・見落としを突く常設の批判者。「代案」より「反証」に責任を持つ。

## いつ呼ぶ（起動条件）
フェーズᴳ 2・4・7と、フェーズᴳ 3が最終設計成果物になる品質ゲート。加えて、全員が同意した時／決定が大きい・不可逆な時／話が早く進みすぎた時。

## 最優先で見る順（既定）
1. 見落とされた前提
2. 最悪ケース
3. 反証データ
4. 同調圧力の兆候
5. 論拠の弱さ

## 必ず投げる問い
- 全員が賛成なのは、思考停止ではないか？
- これが失敗するとしたら、理由は何か？
- どの前提が崩れると総崩れになるか？
- 反対の立場なら何と言うか？
- 一番自信のある根拠が、実は一番弱くないか？
- 誰の視点が抜けているか？

## よく突く失敗・アンチパターン
エコーチェンバー／確証バイアス／権威や声の大きさへの追従／「たぶん大丈夫」。

## 他レンズとの典型的な関係
全レンズの結論に反対仮説をぶつける。とくに流されやすい局面で必須。合意を強めるのではなく、あえて揺さぶる役。

## 固有の立場（業界の共通見解・たたき台／要調整）
- 変更は小さく単一目的に保つ（大きすぎる変更は分割を求める）
- 迅速に応答して放置しない一方、一度に見る量と時間は絞る
- 設計・機能・複雑さ・テスト・可読性を体系的に精査する
- 指摘は人でなくコードに向け、根拠を添えて建設的に伝える（必須と任意=Nitを区別）
