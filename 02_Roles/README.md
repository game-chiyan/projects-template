# 02_Roles — 判断のための観点レンズ集

## これは何か
判断を複数の専門家の観点から見るためのレンズ集。各 md は「その役割が何を大事にし、何を問うか」を薄く載せたもの。**決定者ではなく助言**。最終判断は `_judgment-axis.md`（判断軸）に従って自分で下す。

## 使い方（プロトコル）
- 重要な判断では、関連する 2〜3 レンズ＋必ず `reviewer`（批判者）を併用する。ひとつの声だけで決めない。
- レンズは断定でなく**トレードオフ**で答える。意見が割れて当然。
- 割れたら `_judgment-axis.md` に立ち返る。それでも決まらなければ判断軸末尾の手続きへ。

## 各レンズの出力の型（共通）
1. このレンズの結論（一言）
2. 根拠になる問い・観点
3. リスク・見落とし
4. 他レンズと対立しそうな点

## レンズ一覧（起動条件つき）

| ファイル | 役割 | いつ呼ぶ |
|---|---|---|
| po.md | PO・事業価値 | なぜ作るか／価値・優先度／やらない判断 |
| pm.md | PM | 進行・スコープ・段取り・リスク |
| architect.md | 設計者 | 実装方式・技術選定・境界設計・構造改修 |
| coder.md | コーダー | 実装の書き方・可読性・保守・工数感 |
| qa.md | 品質・テスト | 出荷判断・壊し方・異常系・予防 |
| ux-designer.md | デザイナー（UX） | 利用者・フロー・使いやすさ |
| security.md | セキュリティ | 本番・データ・外部公開・攻撃面 |
| sre.md | 運用・SRE | 稼働・障害・監視・復旧 |
| reviewer.md | レビュワー（批判者） | 総意にわざと反対し盲点を突く（常用） |
| beginner.md | 初学者 | 可読性・学習しやすさ・暗黙知の炙り出し |

全 10 レンズ作成済み。判断軸は設定済み。各レンズの「固有の立場」は業界の共通見解をたたき台として記入済み（出典は `SOURCES.md`）。実運用で自分・組織の方針に上書きしていく。

補助ファイル：`_judgment-axis.md`（判断軸）、`SOURCES.md`（固有の立場の出典）。運用は `CROSS_PROJECT_RULES.md` の「判断レンズ運用」に規定。

## 更新規律
