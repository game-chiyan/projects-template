# 02_Roles — 判断のための観点レンズ集

## これは何か
判断を複数の専門家の観点から見るためのレンズ集。各mdは「その役割が何を大事にし、何を問うか」を薄く載せたもの。**決定者ではなく助言**。最終判断は `_judgment-axis.md`（判断軸）に従い、依頼者または依頼者が指定した承認者が下す。

## 使い方（プロトコル）
- フェーズᴳ 2・4・7と、フェーズᴳ 3が最終設計成果物になる品質ゲートでは `reviewer`ᴳ（批判者）を用いる。
- 高リスクな判断では、関連する 2〜3 レンズ＋ `reviewer`ᴳを併用する。通常の局所判断では複数レンズを必須としない。
- 高リスクとは、不可逆・対外反映・本番／機密データ・安全／法令・広い影響範囲・リリース可否に関わる判断を指す。
- レンズは断定でなく**トレードオフ**で答える。意見が割れて当然。
- 割れたら `_judgment-axis.md` に立ち返る。それでも決まらなければ判断軸末尾の手続きへ。

## 各レンズの出力の型（共通）
1. このレンズの結論（一言）
2. 根拠になる問い・観点
3. リスク・見落とし
4. 他レンズと対立しそうな点

### reviewerᴳの品質ゲート監査

対象となる品質ゲートでは、通常の4項目に加えて指摘を次の表で追跡する。指摘は根拠、対応、状態、裁定者まで記録し、未対応の必須指摘を残したまま品質ゲートを完了しない。

| ファイル | 役割 | いつ呼ぶ |

全 10 レンズ作成済み。判断軸は設定済み。各レンズの「固有の立場」は業界の共通見解をたたき台として記入済み（出典は `SOURCES.md`）。実運用で自分・組織の方針に上書きしていく。

補助ファイル：`_judgment-axis.md`（判断軸）、`SOURCES.md`（固有の立場の出典）。運用は CR-030〜CR-035 に規定。
